Project #26: IR Obstacle Detector

Use infrared sensors for simple obstacle detection! IR obstacle sensors are perfect for line-following robots, edge detection, and proximity switches.

1. Concept and Description

In this project, we will explore the following concepts:

  • How IR proximity sensors work
  • Digital vs. analog output sensors
  • Adjusting detection threshold
  • Applications in robotics

4. The Code (Arduino Sketch)

Copy and paste the code below into your Arduino IDE.

/*
 * Project #26: IR Obstacle Detector
 * The Arduino Student Lab
 * 
 * Detects obstacles using IR proximity sensor
 * Provides LED and buzzer feedback
 */

// ============================================
// PIN DEFINITIONS
// ============================================
const int IR_SENSOR_PIN = 2;   // Digital input from sensor
const int LED_PIN = 13;        // Visual indicator
const int BUZZER_PIN = 8;      // Audio alert

// ============================================
// STATE TRACKING
// ============================================
bool lastState = false;
unsigned long detectionCount = 0;

// ============================================
// SETUP
// ============================================
void setup() {
  pinMode(IR_SENSOR_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  
  Serial.begin(9600);
  Serial.println("=== IR OBSTACLE DETECTOR ===");
  Serial.println("Adjust potentiometer on sensor to set range");
  Serial.println("");
}

// ============================================
// MAIN LOOP
// ============================================
void loop() {
  // Read IR sensor (LOW = obstacle detected)
  bool obstacleDetected = (digitalRead(IR_SENSOR_PIN) == LOW);
  
  // Update outputs based on detection
  if (obstacleDetected) {
    digitalWrite(LED_PIN, HIGH);    // LED on
    tone(BUZZER_PIN, 1000);         // Buzzer on
    
    // Only print on state change (to avoid flooding serial)
    if (!lastState) {
      detectionCount++;
      Serial.print("OBSTACLE DETECTED! (#");
      Serial.print(detectionCount);
      Serial.println(")");
    }
  } else {
    digitalWrite(LED_PIN, LOW);     // LED off
    noTone(BUZZER_PIN);             // Buzzer off
    
    if (lastState) {
      Serial.println("Clear.");
      Serial.println("");
    }
  }
  
  lastState = obstacleDetected;
  delay(50);
}

/*
 * HOW IR OBSTACLE SENSORS WORK
 * 
 * Components inside the module:
 * +---------------------------+
 * |  [IR LED]    [IR Receiver]|
 * |     ‚óØ           ‚óØ        |
 * |                           |
 * |  [Comparator IC]          |
 * |  [Potentiometer]          |
 * +---------------------------+
 *        |  |  |
 *       VCC OUT GND
 * 
 * Working principle:
 * 1. IR LED constantly emits infrared light
 * 2. When object is near, IR reflects back
 * 3. IR receiver detects the reflection
 * 4. Comparator compares signal vs threshold
 * 5. Output goes LOW when obstacle detected
 * 
 * ADJUSTING THE SENSOR:
 * - Turn potentiometer to adjust detection distance
 * - Clockwise: Increase range (more sensitive)
 * - Counter-clockwise: Decrease range
 * - Typical range: 2cm to 30cm
 * 
 * IMPORTANT NOTES:
 * - IR sensors can be affected by sunlight
 * - Dark surfaces absorb IR, reducing range
 * - Shiny surfaces can cause false triggers
 * - Use multiple sensors for wide coverage
 * 
 * OUTPUT LOGIC (varies by module):
 * - Most modules: LOW = obstacle, HIGH = clear
 * - Some modules are reversed, check yours!
 * 
 * APPLICATIONS:
 * - Robot obstacle avoidance
 * - Line following (detect dark line)
 * - Edge detection (table edge = no reflection)
 * - Touch-free switches
 * - Object counters
 */

  1. IR Emission: Module continuously emits infrared light.
  2. Reflection: Nearby objects reflect IR back to receiver.
  3. Comparison: Signal compared against threshold (potentiometer).
  4. Digital Output: LOW when obstacle within range, HIGH otherwise.
