Project #13: LM35 Temperature Monitor

Read real temperature values using the LM35 analog sensor! This classic sensor outputs a voltage directly proportional to Celsius temperature.

1. Concept and Description

In this project, we will explore the following concepts:

  • Using analog temperature sensors
  • Converting analog readings to real-world units
  • Working with voltage-to-temperature relationships
  • Serial Monitor for data display

2. Components Needed

  • (1) Arduino Uno
  • (1) LM35 Temperature Sensor

/*
 * Project #13: LM35 Temperature Monitor
 * The Arduino Student Lab
 * 
 * Reads temperature from LM35 sensor and displays
 * in Celsius and Fahrenheit on Serial Monitor
 */

// ============================================
// PIN DEFINITIONS
// ============================================
const int LM35_PIN = A0;   // Analog input for LM35

// ============================================
// CONFIGURATION
// ============================================
const float VOLTAGE_REF = 5.0;      // Arduino reference voltage
const int ADC_RESOLUTION = 1024;    // 10-bit ADC (0-1023)
const int READINGS_TO_AVERAGE = 10; // Smooth readings

// ============================================
// SETUP - Runs once at startup
// ============================================
void setup() {
  Serial.begin(9600);
  Serial.println("=== LM35 TEMPERATURE MONITOR ===");
  Serial.println("Reading temperature every second...");
  Serial.println("");
}

// ============================================
// MAIN LOOP - Runs repeatedly
// ============================================
void loop() {
  // Take multiple readings and average them
  float totalReading = 0;
  for (int i = 0; i < READINGS_TO_AVERAGE; i++) {
    totalReading += analogRead(LM35_PIN);
    delay(10);  // Small delay between readings
  }
  float averageReading = totalReading / READINGS_TO_AVERAGE;
  
  // Convert analog reading to voltage
  // Formula: voltage = (reading / 1024) * 5.0
  float voltage = (averageReading / ADC_RESOLUTION) * VOLTAGE_REF;
  
  // Convert voltage to temperature (Celsius)
  // LM35 outputs 10mV per degree Celsius
  // Formula: tempC = voltage * 100
  float temperatureC = voltage * 100.0;
  
  // Convert Celsius to Fahrenheit
  // Formula: tempF = (tempC * 9/5) + 32
  float temperatureF = (temperatureC * 9.0 / 5.0) + 32.0;
  
  // Display results
  Serial.print("Raw ADC: ");
  Serial.print(averageReading, 1);
  
  Serial.print(" | Voltage: ");
  Serial.print(voltage, 3);
  Serial.print("V");
  
  Serial.print(" | Temp: ");
  Serial.print(temperatureC, 1);
  Serial.print("¬∞C / ");
  Serial.print(temperatureF, 1);
  Serial.println("¬∞F");
  
  // Wait before next reading
  delay(1000);
}

/*
 * LM35 TEMPERATURE SENSOR
 * 
 * Key Specifications:
 * - Output: 10mV per ¬∞C (linear)
 * - Range: -55¬∞C to +150¬∞C
 * - Accuracy: ¬±0.5¬∞C (typical)
 * - No external calibration needed
 * 
 * THE MATH EXPLAINED:
 * 
 * 1. ADC Reading (0-1023) represents 0-5V
 *    voltage = (reading / 1024) * 5.0
 * 
 * 2. LM35 outputs 10mV per degree Celsius
 *    At 25¬∞C: output = 25 * 10mV = 250mV = 0.25V
 *    At 100¬∞C: output = 100 * 10mV = 1000mV = 1.0V
 * 
 * 3. Convert voltage to temperature:
 *    tempC = voltage / 0.01 = voltage * 100
 * 
 * EXAMPLE:
 *    ADC reading = 51
 *    voltage = (51 / 1024) * 5.0 = 0.249V
 *    tempC = 0.249 * 100 = 24.9¬∞C ‚úì
 * 
 * TIP: For better resolution, use the internal 1.1V
 * reference with analogReference(INTERNAL) - this
 * gives higher resolution for low temperatures.
 */

5. Explanation & Challenges

üîç How It Works

  1. LM35 Output: Outputs exactly 10mV for every 1¬∞C (e.g., 250mV at 25¬∞C).
  2. ADC Conversion: Arduino's 10-bit ADC converts 0-5V to values 0-1023.
  3. Averaging: Multiple readings are averaged to reduce noise.
  4. Math: Divide by 1024, multiply by reference voltage, multiply by 100.
