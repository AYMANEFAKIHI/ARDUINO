Project #18: Serial Monitor Temperature Logger

Learn the essential debugging skill: Serial communication! Log temperature readings with timestamps to the Serial Monitor - your window into what Arduino is thinking.

1. Concept and Description

In this project, we will explore the following concepts:

  • Serial communication basics
  • Serial.print() vs Serial.println()
  • Formatting data output
  • Creating readable logs with timestamps

2. Components Needed

  • (1) Arduino Uno
  • (1) LM35 Temperature Sensor
  • (1) Breadboard
  • (3) Jumper Wires

3. Physical Montage (Wiring Instructions)

Pin Connections:

Arduino PinComponent
5VLM35 VCC (left pin)
A0LM35 VOUT (middle pin)
GNDLM35 GND (right pin)

4. The Code (Arduino Sketch)

Copy and paste the code below into your Arduino IDE.

/*
 * Project #18: Serial Monitor Temperature Logger
 * The Arduino Student Lab
 * 
 * Logs temperature readings with timestamps
 * Demonstrates Serial communication for debugging
 */

// ============================================
// PIN DEFINITIONS
// ============================================
const int TEMP_PIN = A0;

// ============================================
// LOGGING CONFIGURATION
// ============================================
const unsigned long LOG_INTERVAL = 2000;  // Log every 2 seconds
unsigned long lastLogTime = 0;
unsigned long readingNumber = 0;

// ============================================
// SETUP - Runs once at startup
// ============================================
void setup() {
  // Initialize Serial at 9600 baud
  Serial.begin(9600);
  
  // Wait for Serial to connect (important for some boards)
  while (!Serial) {
    ; // Wait
  }
  
  // Print header
  Serial.println("==========================================");
  Serial.println("     TEMPERATURE DATA LOGGER");
  Serial.println("==========================================");
  Serial.println("");
  Serial.println("Reading#  Time(s)   Raw    Voltage   Temp(C)  Temp(F)");
  Serial.println("--------  -------   ---    -------   -------  -------");
}

// ============================================
// MAIN LOOP - Runs repeatedly
// ============================================
void loop() {
  unsigned long currentTime = millis();
  
  // Check if it's time to log
  if (currentTime - lastLogTime >= LOG_INTERVAL) {
    lastLogTime = currentTime;
    readingNumber++;
    
    // Take temperature reading
    int rawValue = analogRead(TEMP_PIN);
    float voltage = (rawValue / 1024.0) * 5.0;
    float tempC = voltage * 100.0;
    float tempF = (tempC * 9.0 / 5.0) + 32.0;
    
    // Calculate runtime in seconds
    float runtimeSeconds = currentTime / 1000.0;
    
    // Log data in formatted columns
    // Using print() for same line, println() for new line
    
    // Reading number (padded)
    Serial.print("#");
    if (readingNumber < 10) Serial.print("  ");
    else if (readingNumber < 100) Serial.print(" ");
    Serial.print(readingNumber);
    Serial.print("     ");
    
    // Time in seconds
    Serial.print(runtimeSeconds, 1);
    Serial.print("s    ");
    
    // Raw ADC value
    if (rawValue < 100) Serial.print(" ");
    if (rawValue < 10) Serial.print(" ");
    Serial.print(rawValue);
    Serial.print("    ");
    
    // Voltage
    Serial.print(voltage, 3);
    Serial.print("V    ");
    
    // Temperature Celsius
    Serial.print(tempC, 1);
    Serial.print("C    ");
    
    // Temperature Fahrenheit
    Serial.print(tempF, 1);
    Serial.println("F");
    
    // Every 10 readings, add a separator
    if (readingNumber % 10 == 0) {
      Serial.println("--------------------------------------------------");
    }
  }
}

/*
 * SERIAL COMMUNICATION BASICS
 * 
 * Serial.begin(9600):
 *   Initialize serial port at 9600 bits/second
 *   Common rates: 9600, 115200, 250000
 *   Must match Serial Monitor setting!
 * 
 * Serial.print(value):
 *   Send data WITHOUT newline
 *   Can chain multiple prints on same line
 * 
 * Serial.println(value):
 *   Send data WITH newline
 *   Next output starts on new line
 * 
 * Serial.print(float, decimals):
 *   Control decimal places: Serial.print(3.14159, 2) ‚Üí "3.14"
 * 
 * WHY 9600 BAUD?
 *   Historical default, very reliable
 *   For faster data: use 115200 or higher
 *   Ensure both Arduino code AND Serial Monitor match!
 * 
 * COPYING DATA:
 *   You can select and copy text from Serial Monitor
 *   Paste into Excel/Sheets for analysis!
 */

5. Explanation & Challenges

üîç How It Works

  1. Serial.begin(): Initializes communication at specified baud rate.
  2. print() vs println(): print stays on same line, println adds newline.
  3. Formatting: Padding and alignment make data readable.
  4. Timestamps: millis() provides runtime in milliseconds.

üöÄ Challenges

  • CSV Format: Output comma-separated values for easy spreadsheet import.
  • Alerts: Print "WARNING" when temperature exceeds a threshold.
  • Statistics: Calculate and display min/max/average every 10 readings.
  • Multiple Sensors: Log from 2-3 temperature sensors simultaneously.
