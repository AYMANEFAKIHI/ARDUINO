Project #10: Vibration Sensor (SW-420) Alert

Detect vibrations, knocks, and impacts with the SW-420 sensor! Perfect for building earthquake detectors, knock sensors, and anti-theft alarms.

1. Concept and Description

In this project, we will explore the following concepts:

  • Using vibration/shock sensors
  • Adjusting sensor sensitivity
  • Building impact detection systems
  • Counting vibration events

4. The Code (Arduino Sketch)

Copy and paste the code below into your Arduino IDE.

/*
 * Project #10: Vibration Sensor (SW-420) Alert
 * The Arduino Student Lab
 * 
 * Detects vibrations, knocks, and impacts
 * Counts events and triggers alerts
 */

// ============================================
// PIN DEFINITIONS
// ============================================
const int VIBRATION_PIN = 2;   // Vibration sensor digital output
const int LED_PIN = 13;        // Alert LED
const int BUZZER_PIN = 8;      // Alert buzzer (optional)

// ============================================
// CONFIGURATION
// ============================================
const unsigned long ALERT_DURATION = 500;    // How long LED stays on (ms)
const unsigned long DEBOUNCE_TIME = 50;      // Ignore rapid triggers (ms)

// ============================================
// STATE VARIABLES
// ============================================
int vibrationCount = 0;              // Total vibrations detected
unsigned long lastVibrationTime = 0; // For debouncing
unsigned long alertStartTime = 0;    // When alert began
bool alertActive = false;            // Is alert LED on?

// ============================================
// SETUP - Runs once at startup
// ============================================
void setup() {
  pinMode(VIBRATION_PIN, INPUT);
  pinMode(LED_PIN, OUTPUT);
  pinMode(BUZZER_PIN, OUTPUT);
  
  Serial.begin(9600);
  Serial.println("=== VIBRATION SENSOR READY ===");
  Serial.println("Tap, knock, or shake the sensor!");
  Serial.println("Adjust the potentiometer on the module");
  Serial.println("to change sensitivity.");
  Serial.println("");
}

// ============================================
// MAIN LOOP - Runs repeatedly
// ============================================
void loop() {
  unsigned long currentTime = millis();
  
  // Read vibration sensor
  // LOW when vibration detected (module pulls low)
  bool vibrationDetected = (digitalRead(VIBRATION_PIN) == LOW);
  
  // Check for new vibration (with debounce)
  if (vibrationDetected && (currentTime - lastVibrationTime > DEBOUNCE_TIME)) {
    lastVibrationTime = currentTime;
    
    // Increment counter
    vibrationCount++;
    
    // Start alert
    alertActive = true;
    alertStartTime = currentTime;
    digitalWrite(LED_PIN, HIGH);
    tone(BUZZER_PIN, 1000, 100);  // Short beep
    
    // Report to Serial
    Serial.print("VIBRATION #");
    Serial.print(vibrationCount);
    Serial.println(" detected!");
  }
  
  // Turn off alert after duration
  if (alertActive && (currentTime - alertStartTime >= ALERT_DURATION)) {
    alertActive = false;
    digitalWrite(LED_PIN, LOW);
  }
  
  // Every 10 seconds, report total count
  static unsigned long lastReportTime = 0;
  if (currentTime - lastReportTime >= 10000) {
    lastReportTime = currentTime;
    Serial.print("-- Total vibrations in last 10s: ");
    static int lastCount = 0;
    Serial.println(vibrationCount - lastCount);
    lastCount = vibrationCount;
  }
}

/*
 * SW-420 VIBRATION SENSOR MODULE
 * 
 * The module contains:
 * - SW-420 vibration switch (spring-based)
 * - Comparator circuit
 * - Sensitivity potentiometer
 * - LED indicator
 * 
 * How it works:
 * - A spring inside vibrates when disturbed
 * - Spring touching contacts = circuit closes
 * - Comparator outputs digital HIGH/LOW
 * 
 * Adjusting sensitivity:
 * - Turn potentiometer clockwise = MORE sensitive
 * - Turn counter-clockwise = LESS sensitive
 * - Adjust until LED triggers at desired force
 * 
 * APPLICATIONS:
 * - Anti-theft alarm (car, bike, laptop)
 * - Earthquake early warning
 * - Knock detection (secret knock door lock)
 * - Machine monitoring (detect malfunction)
 */

5. Explanation & Challenges

üîç How It Works

  1. Spring Mechanism: A conductive spring inside vibrates when disturbed.
  2. Digital Output: Module outputs LOW when vibration exceeds threshold.
  3. Sensitivity Adjustment: Potentiometer on module sets the trigger threshold.
  4. Event Counting: Code tracks total vibrations and reports periodically.
  5. Debouncing: Ignores rapid multiple triggers from single vibration.

üöÄ Challenges

  • Knock Pattern: Detect a specific knock pattern (like shave-and-a-haircut) to unlock.
  • Intensity Levels: Use the analog output (if available) to measure vibration strength.
  • Data Logger: Record timestamps of all vibrations to SD card.
  • Seismograph: Plot vibration intensity over time using Serial Plotter.
