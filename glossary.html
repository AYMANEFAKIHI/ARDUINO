Quick definitions and syntax references for the most common terms, functions, and components you will encounter in your projects.

1. Core Programming Concepts

Sketch
The name given to an Arduino program file (written in C/C++).
Setup()
A function that runs only **once** when the Arduino board starts up or is reset. Used for initializing pins, libraries, and communication.
Loop()
A function that runs repeatedly forever, cycling as long as the board has power. This contains the main logic of the program.
Variable
A storage location defined in the code, used to hold data (like a sensor reading or a pin number).
Array
A structured variable that can store multiple values of the same type (e.g., storing a list of all 6 LED pin numbers for the Knight Rider project).
If/Else
A control structure that allows the program to make decisions (e.g., "IF the button is pressed, THEN turn the LED ON, ELSE turn the LED OFF").
Function
A block of code given a name that performs a specific, reusable task (e.g., a function named `allOff()` to turn all traffic lights off).
FSM (Finite State Machine)
A programming model where the system can only be in one "state" at a time (e.g., RED, YELLOW, or GREEN) and transitions between them based on conditions.

2. Basic Electronics & Hardware

Breadboard
A construction base used to build temporary, solderless electronic circuits by plugging in components and wires.
GND
Ground (0 Volts). The reference point for all voltage measurements in a circuit.
5V / VCC
The standard 5-Volt power supply output from the Arduino board, used to power small components.
Resistor
A passive component that limits the flow of electric current. Essential for protecting sensitive parts like LEDs.
Pull-Up / Pull-Down Resistor
A resistor used with input pins (like buttons) to hold the pin stably at HIGH (Pull-Up) or LOW (Pull-Down) when the main input is disconnected, preventing a "floating" or noisy signal.
LED
Light Emitting Diode. An electronic component that emits light when current flows through it. It must be connected with correct polarity (+Anode, -Cathode).

3. Input/Output Modes & Core Functions

Digital I/O
A signal that is strictly **ON** (HIGH/5V) or **OFF** (LOW/0V). Used for pins labeled D0-D13.
Analog I/O
A signal that can represent a range of values (variable). Used for Analog Pins (A0-A5).
PWM (Pulse Width Modulation)
A technique used to simulate analog output (like fading an LED or controlling motor speed) by rapidly switching a digital pin ON and OFF. Pins are marked with a (~).
Interrupt
A hardware mechanism that allows the Arduino to temporarily stop what it's doing in the main `loop()` to immediately execute a high-priority task (e.g., a fast button press).

4. Essential Function Cheatsheet

FunctionPurposeSyntax
pinMode() Sets a pin to behave as an INPUT or an OUTPUT. pinMode(pin, mode);
digitalWrite() Sets a digital output pin HIGH (ON) or LOW (OFF). digitalWrite(pin, value);
digitalRead() Reads the value of a digital input pin (returns HIGH or LOW). digitalRead(pin);
analogRead() Reads a voltage from an Analog pin (A0-A5) and returns a value between 0 and 1023. analogRead(pin);
analogWrite() Writes an analog value (PWM) to a PWM pin (~) using a value between 0 and 255. analogWrite(pin, value);
delay() Pauses the program for a specified number of milliseconds (ms). (Blocks code execution) delay(ms);
millis() Returns the number of milliseconds since the program started. Used for non-blocking timing. unsigned long time = millis();
map() Scales a number from one range of values to another (e.g., 0-1023 to 0-255). map(value, fromL, fromH, toL, toH);
pulseIn() Measures the duration of a pulse (HIGH or LOW) on a specified pin. Used for sensors like HC-SR04. time = pulseIn(pin, value);

5. Communication Protocols

Serial (UART)
The simplest form of two-way communication using only two wires (RX and TX). Used for debugging via the Serial Monitor and communicating with modules like GPS or Bluetooth (HC-05).
I2C (Inter-Integrated Circuit)
A powerful two-wire bus protocol (SDA, SCL) that allows the Arduino to communicate with many devices (OLED displays, RTCs, MPU-6050) using unique addresses. Very common in advanced projects.
SPI (Serial Peripheral Interface)
A fast communication protocol that uses four wires (MISO, MOSI, SCK, CS). Often used for high-speed peripherals like SD cards and RFID readers.
Baud Rate
The speed at which data is transmitted over a serial line, measured in bits per second (bps). Must match between the Arduino and the connected device (e.g., 9600).
